Render · Option A 路线图

让 opencli 的整条生命周期跑在 Render 自己的浏览器里

分支 feat/opencli-bridge · 每个里程碑由独立 QA 对抗式验收 · 监督循环跑到"完全自主"为止

北极星:Render 是一个自包含的 Chromium 浏览器。Agent 的所有浏览器动作(登录、cookie 抓取、导航、截图…)都由 Render 自己充当 opencli 的浏览器后端——Render 主进程接 opencli daemon 的 /ext,用自己的 webContents.debugger(CDP)驱动自己的 WebContentsView。
做法不是"给 Render 装插件系统"(Electron 结构上跑不了依赖 chrome.debugger 的扩展,做完整插件系统=维护 Chromium fork,见架构决策文档),而是反过来:让 opencli 连到 Render。
铁律(贯穿所有里程碑):任何代码、任何测试,都绝不驱动 / 打开 / 关闭你的系统 Chrome。 全程只用 Render 自己的 render profile。验证"共存"只查 daemon 状态,不真跑 Chrome 搜索。M1 那次"关 Chrome"是错误捷径,已在 M1.5 根除并由 QA 独立证实。
