The decisive proof

The question the final eval answers: does a real prompt produce a real codex turn that uses real hands and renders through json-render, with a working human round-trip? The killer test:

An approved shell command actually ran in the sandbox and the agent used its real output. Agent reported: date -u → Mon Jun 22 04:29:50 UTC 2026 Wall clock (s): Mon Jun 22 04:30:26 UTC 2026 ⇒ 36 seconds apart A mock or hallucination cannot match the machine's clock. The command genuinely executed behind a human approval.

What runs, hand by hand

Brain

codex app-server (gpt-5.4) driven over JSON-RPC-lite/stdio, spawned inside the sandbox. Real streamed turns (44 deltas in the proof).

App hand

opencli router: public adapters → sandbox (3 real arXiv papers returned), browser adapters → CDP human-hand. login() journey wired.

Sandbox hand

One SandboxProvider seam: local-seatbelt (macOS sandbox-exec, keyless, default) + e2b drop-in when E2B_API_KEY is set.

Human hand

In-process Chromium via webContents.debugger (CDP 1.3): attaches, round-trips document.title, exposes a relay endpoint (http://127.0.0.1:63958 in the proof).

Two real integration findings (fixed at source)

  • codex 0.136.0 approval decisions are accept/cancel, not approved/denied — the wrong value silently fails the command. Mapped allow→accept / deny→cancel.
  • Local ~/.codex PermissionRequest hooks swallow approvals before the protocol, stalling HITL. Render launches codex against a hook-free CODEX_HOME so approvals flow to the ux-confirm panel.

Scope & honest caveats

  • This is the pure-browser desktop prototype (macOS/Linux via Electron), as scoped. iOS/Android are future thin clients that reuse the same @render/protocol contracts against a remote sandbox.
  • e2b is implemented behind the seam but exercised as a guarded skip without E2B_API_KEY; set a key to validate the cross-end brain.
  • GUI screenshot of the live Electron window was blocked by macOS screen-recording permission; the journey is proven by the headless proof using identical wiring.
  • Hardening backlog: CDP action-scoping (task_id + allowed_origins + TTL), per-account profile isolation, tunnel auth for remote client-pull.
