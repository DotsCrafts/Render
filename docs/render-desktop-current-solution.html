维度现状(③)Option-B 目标
页面来源agent 手写 HTML + Render createArtifactopencli ux serve 产出并常驻服务,返回 {url,session}
取数window.renderArtifact.opencli → Render IPC页面 → opencli POST /ux/data(token+精确Origin)
并发无队列 → 撞死/ux/data FIFO 并发=1 + agg --concurrency 1
隔离Render CSP + artifact:<id> 分区opencli 独立 origin + 一次性 token + served CSP
Render 角色实现一套平行执行/会话/能力纯浏览器:render-open <url>
先补后拆(codex 调研结论):① ux render --no-open 现在留不下活 URL(8s 退出,--keep 未透出)——先加 opencli ux serve;② daemon 不串行(bridge 帧并发,无 mutex)——/ux/data 必须自带 FIFO 队列;③ 安全升级:精确 Origin、served CSP、服务端 allowlist、写操作(add-cart 等)默认拒。跑通一个真能搜的活页面后,才删 Render 旧桥。
