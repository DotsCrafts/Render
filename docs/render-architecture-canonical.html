Render — 整体架构(权威)

Electron + 移动原生 的 agent 浏览器 · codex App Server 单协议 · 复用 krew-dev(e2b/krewhub/krewauth/agent-vault)· 双平面凭证
锁定的决策
  • 大脑协议 = codex App Server(单选)。流式/审批/HITL/线程全保真;Render 已建于此。不用 MCP/ACP/opencli-adaptor 当主协议。
  • 大脑位置:桌面本地 seatbelt(零成本)/ 桌面远端 + 移动 = e2b sandbox(复用 krew 已部署集群)。
  • 跨端传输:App Server 的 stdio 帧 —— 桌面本地走管道、桌面远端走 e2b SDK、移动走 krewhub WS 网关。同一批帧。
  • Plane-2(网站登录态)永不出设备;Plane-1(模型凭证)设备保管(OAuth refresh_token 留设备,仅发短时 token;或 egress 经设备)。
  • 浏览器驱动:opencli /ext 会合(设备拨出 + daemon 路由,NAT 友好);后端可插拔 —— 桌面 CDP、移动原生 WebView。
