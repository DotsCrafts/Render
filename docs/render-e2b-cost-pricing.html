自建 e2b 成本 & 用户定价测算

Render 远端大脑(codex app-server in sandbox)的运营成本与收费模型 · 数字为估算,可在下方计算器里调成你的真实参数

先记住三个反直觉的事实

  • 桌面端运营成本 = $0。 mac/win 的大脑跑 local-seatbelt 在用户自己机器上,你不掏一分钱算力。成本只发生在移动端(必须远端大脑),或你主动提供"云端大脑"选项时。
  • 模型推理费 ≫ sandbox 算力费,通常 10–50 倍。 一个 agent 编码会话烧 10 万–100 万 token;sandbox 那点 CPU 是零头。真正的成本杠杆是模型,不是 e2b。
  • 用户自带 key(BYO-key)能把你的成本打到几乎为零。 用户出模型钱,你只承担 sandbox 算力(每会话几分钱)。这决定了定价能不能做"近免费"。

3 · 收费模型建议

层谁付模型你的成本定价
Free / 桌面BYO-key$0(本地大脑)免费。获客 + 留存。你现在的 ifunk key 就是这模式。
Pro · BYO-key用户自带仅 sandbox(每会话几分钱)$5–9/mo 订阅。覆盖远端大脑 + 移动端;毛利极高。推荐主力。
Managed你代付模型主导($0.1–3/会话)用量计费 或 高价订阅(~$20–40/mo,带额度)。必须盯模型成本。
结论: 自建 e2b 的"算力账"很便宜(每会话几分钱,自建在中等并发以上比官方省)。真正决定能不能赚钱的是"模型谁付" —— 主推 BYO-key:用户出模型钱,你收一个低订阅费覆盖远端大脑,毛利稳。Managed 作为高端可选,定价必须把模型成本算进去。桌面端永远免费(本地大脑零成本),把它当获客漏斗顶。

4 · 自建 vs 官方 e2b:什么时候切自建

  • 启动期 / 低并发:用官方 e2b(或本地 + 少量移动用户)。省运维,固定成本低,$150/mo 起。
  • 规模期:月 sandbox 小时数大到官方用量费 > 自建(控制面 + 节点)时,切自建(Hetzner 裸金属)。计算器里的 "vs 官方 e2b" 那行就是 break-even 信号:它比"基础设施 $/mo"高出越多,自建越值。
  • 自建硬门槛:Firecracker 要 KVM 嵌套虚拟化 → 需裸金属 / 支持嵌套虚拟的实例 + e2b 自托管运维(Nomad/orchestrator)。这是隐性的"人力成本",启动期不划算。
