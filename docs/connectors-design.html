Render Connectors

把所有 opencli site adapters 的登录态做成 Manus Connector 式的产品心智：站点即服务、状态即真相、登录即恢复。

1 · 产品心智

站点即服务每个 opencli adapter 是一个可连接的服务卡片：图标 + 名称 + 域名 + 命令数。公共（无需登录）adapter 折叠为底部 chips，目录完整可见。
状态即真相状态唯一来源是 opencli <site> whoami --site-session persistent。缓存只作首屏秒开的提示（userData/connectors.json），过期即惰性重探。
登录即恢复Connect 在 Render 内打开站点登录页，whoami watch 轮询检测；登录落地那一刻卡片自动翻转为 Connected，对话若在等待则被 steer 恢复 —— 用户永远不用回答"登录好了吗"。

状态语义（badge）

状态含义动作
Connectedwhoami 验证过的活跃会话（显示账号名）Disconnect · Check
Not connectedexit 77 / logged_in:false —— 权威的"未登录"Connect · Check
Waiting for sign-in登录页已打开，watch 轮询中（≈4 分钟上限，绝不永久转圈）Check
Checkingwhoami 探测进行中—
Unknown未探测过 / 引擎不可用 / 超时 —— 诚实的不确定，附 detail 说明Connect · Check

2 · whoami 真值表（opencli 1.8.4 契约）

信号判定理由
exit 0 + logged_in:trueconnected（含账号）权威成功
exit 77 / logged_in:falsedisconnectedauth gate 是权威的"未登录"信号
非零 + COMMAND_EXEC（校验漂移）connected + detailauth gate 在命令体之前运行 —— 能失败在命令体里，说明会话是活的（如 dianping "member page rendered but no user_id link found"）
exit 69 / 124 / 127 / CLI 拒绝 / 其它引擎错误unknown + detail冒烟实测修正：OpenCLIApp 的 exit-78 环境漂移曾把全部站点误判为 Connected。引擎级失败绝不能变成登录verdict —— unknown 诚实且可恢复
关键教训：漂移即已登录的规则必须要求 COMMAND_EXEC 的正向证据；默认分支是 unknown，不是 connected。宁可"不知道"，不可撒谎。

3 · 状态机与登录旅程

  • watch：12s × 20 次（≈4 分钟）轮询 whoami；命中 connected 即停 → 更新卡片 + 持久化 + onConnected；耗尽落回最后的稳定态并给出提示，绝不留下永久 spinner。
  • generation 取消语义：每个 watch 持有代号，新的 connect / disconnect / dispose 令旧循环安静退场。
  • 持久化边界：只有 connected / disconnected / unknown 落盘；checking / connecting 是纯运行时状态。

任务中断 → 自动恢复（完整旅程）

agent 命令 exit 77 → login 卡片（含"我会自动检测"文案）→ 用户点 Sign in
→ Render 打开登录 tab（cookie 落在 persist:render + persistent site-session）→ noteLoginOpened 启动 watch
→ 用户扫码/输密码（Render 永远看不到凭据）→ whoami 翻转 connected
→ notifyLogin：live turn 被 steer（"session is live, retry with --site-session persistent"）自动重试；无 live turn 则发一张"已连接，说继续"卡片 —— 绝不擅自开新回合

5 · 验证

  • 单元：opencli-router 29/29（whoami 真值表、argv 契约、目录聚合、login unbounded）；ConnectorService 13/13（缓存合并、staleness 门、watch 翻转/耗尽、logout 不支持的诚实降级、快照流）。
  • e2e（真实 Electron）：nav-guard 通过；connectors 冒烟脚本走通 renderer→preload→main→router 全链路 —— 点击顶栏按钮、面板挂载、缓存站点渲染、探测后 badge 更新。
  • 冒烟抓到并修复的真 bug：本机 OpenCLIApp 环境漂移（exit 78）曾让三张卡片全部假显 Connected —— 收紧漂移规则后诚实显示 Unknown + detail。
  • 已知环境问题（另案处理）：本机 OpenCLIApp 向其自带 CLI 注入已废弃的 OPENCLI_DAEMON_PORT，所有 opencli 调用报错 —— 影响整个 app hand，非本特性引入。
